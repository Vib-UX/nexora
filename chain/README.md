# chain/

Orbit chain config + scripts for **Nexora Devnet** (chainId `20056` / `0x4E58`).

## Files

| File | Purpose |
|---|---|
| `orbit-config.json` | Canonical chain definition (consumed by deploy script + dashboard) |
| `deploy-l3.ts` | `plan` / `local` / `execute` modes for Orbit deployment |
| `docker-compose.yml` | Local Nitro node + optional Blockscout explorer (`--profile explorer`) |
| `.env.example` | Tunables for the Blockscout explorer profile (copy to `.env`) |
| `blockscout/proxy.conf.template` | nginx template mounted into the internal `blockscout-proxy` container |
| `data/nitro-node-config.json` | Generated by `mode=local`, mounted into Nitro container |

## Modes

```bash
# 1. Print plan only (no side effects)
pnpm --filter @nexora/chain deploy:l3

# 2. Generate a local nitro-node config for docker-compose
ORBIT_DEPLOY_MODE=local pnpm --filter @nexora/chain deploy:l3

# 3. Deploy the rollup contracts on parent chain (requires .env)
ORBIT_DEPLOY_MODE=execute pnpm --filter @nexora/chain deploy:l3
```

## Required environment (`.env` here)

```
PARENT_RPC_URL=https://sepolia-rollup.arbitrum.io/rpc
ORBIT_OWNER_PRIVATE_KEY=0xabc...
SEQUENCER_ADDRESS=0x...
VALIDATOR_ADDRESS=0x...
```

## Boot the local devnet

```bash
ORBIT_DEPLOY_MODE=local pnpm --filter @nexora/chain deploy:l3
docker compose up -d nitro
# optional Blockscout explorer (full v9 stack, ~1 GB RAM)
# cp .env.example .env  # tune values; defaults work
docker compose --profile explorer up -d
docker logs -f nexora-blockscout-backend  # ~60s for first migrate
```

Chain state lives in the Docker volume **`nexora-nitro-data`** (not bind-mounted `./data`), so running Compose with `sudo` does not leave root-owned files that block Nitro (`jwtsecret` permission denied). To reset the devnet state: `docker compose down` then `docker volume rm chain_nexora-nitro-data` (prefix may match your compose project name).

If you **bind-mount** `./data` manually, ensure the container user can write it: `sudo chown -R 1000:1000 chain/data`.

RPC: `http://127.0.0.1:8547` · WS: `ws://127.0.0.1:8548` · Explorer (with `--profile explorer`): `http://127.0.0.1:4001`

Ports map to **loopback only** so the node is not exposed on the LAN; use [deploy/vps/](../deploy/vps/README.md) (nginx + TLS) for a public hostname. See [docs/blockscout.md](../docs/blockscout.md) for the full Blockscout service inventory and the lean-stack opt-out.
